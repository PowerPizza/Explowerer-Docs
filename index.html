Understanding Basics

To understand this project and how it works we need to know about the content of all the three folders you got after following installation process.

1. ngrok linker

When the server starts, it launches a web server on localhost and simultaneously uses ngrok to make this web server accessible worldwide via a generated URL. The target device connects to this ngrok URL and communicates with the web server. Each time the server restarts, ngrok generates a new URL. This dynamic URL change disrupts the target's connection, as the hard-coded URL in the target no longer points to the server, causing communication to fail.
To ensure the target can access the server reliably, even if the ngrok URL changes, we use a hosting service as an intermediate step. The hosting service provides a fixed URL, which we embed in the targetâ€™s executable file. This URL remains consistent, ensuring the target can always locate the serverâ€™s updated address.
We have to upload this tool called ngrok linker to the hosting service. Each time the server starts, it sends the updated ngrok URL to this ngrok linker, which stores it.
When the target starts, it contacts the ngrok linker on the hosting service to retrieve the current ngrok URL. If the URL is valid, the target connects to the server. If not, the target retries, asking ngrok linker for an updated URL until a connection is established.
